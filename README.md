# Llama.cpp Windows Desktop

一个 Windows 桌面端控制面板，用于运行和管理本地 llama.cpp 推理服务。内置聊天界面，支持 OpenAI 兼容 API、多模态附件、MCP 工具协议和自定义助手系统。

> 基于 [Qiao-920/llama-cpp-desktop](https://github.com/Qiao-920/llama-cpp-desktop) 进行了大量功能增强和体验优化。

---

## 功能亮点

- **本地直连** — 直接启动 llama.cpp 的 `llama-server.exe`，无需额外启动器
- **OpenAI 兼容** — 默认提供 `http://127.0.0.1:8080/v1`，可接入任何 OpenAI 客户端
- **桌面聊天** — 流式回复、Markdown 渲染、代码块复制/预览、历史对话管理
- **MCP 协议支持** — 接入外部 MCP 工具服务器，模型自动发现并调用工具
- **助手系统** — 创建命名助手并设置系统提示词，一键切换角色
- **附件支持** — 图片、文本、PDF 等附件，图片可在聊天中预览
- **系统托盘** — 关闭窗口后隐藏到托盘，服务继续后台运行

## 相比原项目的改进

### 新增功能
- **MCP 协议支持** — 完整的 JSON-RPC over stdio MCP 客户端，管理外部工具服务器，聊天中自动注入工具列表
- **助手系统** — 创建/切换/管理命名助手，每个助手可配置独立的系统提示词
- **Markdown 渲染** — 自研轻量渲染器，支持标题、粗体、斜体、链接、列表、表格、块引用、代码高亮
- **LaTeX 自动转换** — `\rightarrow` → `→`、`\times` → `×`、下标/括号自动清理
- **清除上下文/删除所有消息** — 消息操作栏新增一键清空和删除按钮
- **重新生成按钮** — 每条回复可基于同一问题重新生成答案

### 安全性修复
- **移除硬编码本机路径** — 不再包含作者本地绝对路径，改为空默认值 + 用户配置
- **`shell.openExternal` 协议白名单** — 只允许 `http(s):` 协议，本地路径走 `shell.openPath`
- **CSP 内容安全策略** — 限制脚本和连接来源
- **大量静默吞错改为日志记录** — `fetchJson`、`readJson` 等关键路径加了错误日志

### 进程管理优化
- **优雅退出** — 先尝试关闭 stdin，3 秒超时后才 `taskkill /F` 强杀
- **`before-quit` 超时保护** — `Promise.race` 限制最多等 5 秒，防止应用卡死无法退出

### 体验优化
- **滚动位置保持** — 复制、切换标签等操作不再导致页面跳回顶部
- **设置面板标签列表滚动保持** — 左侧标签栏切换后不滚回顶部
- **设置面板模型名不遮挡按钮** — 修复了 CSS 绝对定位泄漏问题
- **代码去重** — 消息构建逻辑从两处内联 ~60 行统一为 `prepareChatMessages`
- **TOML 配置边缘处理** — 空路径不再拼出无意义的相对路径

### UI 改进
- 历史对话限制 8 条 + 滚动查看
- 助手标签开关式切换（点击选中，再点取消）
- 对话中切换助手自动注入提示词
- 系统提示词输入框自适应高度

## 快速开始

1. 下载 `Llama.cpp-Desktop.exe`（Releases 页面）
2. 双击运行，首次启动在设置中指定 llama.cpp 目录和模型文件
3. 保存配置并启动服务
4. 使用内置聊天，或将 `http://127.0.0.1:8080/v1` 接入其他客户端

## MCP 使用说明

1. 设置 → MCP 标签页 → 添加 MCP 服务器
2. 输入启动命令，例如：`npx -y @anthropic/mcp-filesystem C:\path\to\dir`
3. 服务器自动连接，聊天时模型将自动获取工具列表并调用
